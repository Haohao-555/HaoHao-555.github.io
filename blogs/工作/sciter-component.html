    # Web 组件库

    # 1、组件库总体架构

    # 2、目录结构及组件库引入

    使用时，只需要引入 CSS 样式文件及组件入口文件即可。

    # 3、组件注意事项

    1、当组件支持绑定事件时，事件名称命名规则为 Sc + DOM事件名称且首字母大写，即点击事件（click）对应（ScClick）

    在 sciter 中，通过原生 DOM 渲染的组件，只有按钮标签才支持获取焦点，失去焦点事件

    2、当绑定事件时，回调函数接收的第一个参数为事件对象（event）

    3、创建并初始化组件会返回该组件的配置信息（对应到组件的 API 为 create+组件名）

    const progressState = new ScProgress().createProgress({ // 配置信息 });
console.log('progress', progressState);
    1
    2

    若：页面没有出现组件，且无报错信息输出，可以打印其返回的信息，大概率是容器找不到或者出现容器覆盖

    4、部分组件有提供生成组件，并将组件的返回给调用方。 （对应到组件的 API 为 get + 组件名）

    const btnlist = new ScButtons().getBtnNodeList([{}, {}, {}]);
console.log(btnlist);
    1
    2

    5、组件唯一的id是绑定在组件的最外层DOM 标签上，即调用方当在某些特殊情况，需要操作组件的DOM是可以通过该id来获取到该组件。获取方式如下：

    document.querySelector(`#${btnlist[0].id}`);
document.querySelector(`#${progressState.id}`);

    3、req 函数的实现

    function req(key) {
    const test_list = [];
    return new Promise((resolve, reject) => {
        let res = [];

        // 模拟延迟 （数据处理）
        setTimeout(() => {
            let dfs = (list, id) => {
                for (let i = 0; i < list.length; i++) {
                    let item = list[i];
                    if (item.id == id) {
                        res = item.children || [];
                        break;
                    } else {
                        if (item.children) dfs(item.children, id);
                    }
                }
            }
           
            // 查找该结点下的子节点
            dfs(test_list, key);
            resolve(res);
        }, 1000);
    })
}

    # 4.6、穿梭器组件

    # 4.6.1、效果图

    # 4.6.2、功能说明

    • 对数据项进行操作
      • 增加：点击增加按钮，可以从左容器中获取到被选中的数据项，并将其添加到右容器中。此时左容器中的数据项为禁止状态
      • 删除：点击删除按钮，可以从右容器中获取到被选中的数据项，并将其添加到左容器中。此时左容器中的数据项后根据右容器传递过来的数据项进行匹配解除禁止状态。
      • 清空：点击清空按钮，可以从右容器中获取所有数据项，并将其添加到左容器中。同理：此时左容器数据项也会解除禁止状态
    • 头部实时记录被选中的列表数据项或叶子数据项的数量
    • 搜索：根据配置参数来确定是否配置搜索功能
      • 搜索方法可以自定义方法，也可以使用默认搜索方法
    • 双击数据项：实现被双击数据项的增加（左容器）或删除（右容器）

    # 4.7.4、配置说明

    字段 value 说明
    container css选择器 DOM 容器
    mulselect true | false 是否配置多选（可选）
    sortFunc Function 排序函数（见注意事项）（可选）

    # 4.7.5、API 说明

    API 参数 说明 返回值
    createTable 见配置说明 生成组件，添加到容器中（提供容器） 返回组件状态（见组件注意事项）
    pushData 见配置说明 添加数据 无返回值

    # 4.7.6、注意事项

    1、若排序函数（sortFunc）未定义，则组件内部会使用内置的排序函数进行排序（字符串排序）

    2、排序函数接收三个参数：

    • 第一个参数为 state：升降序（true：升序 | false：降序）
    • 第二个参数为 param：该字段的表格头部信息（即排序的字段）
    • 第三个参数为 data：当前表格的数据

    3、 pushData 添加的数据不会进行累加
